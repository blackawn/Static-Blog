JavaScript基本类型和对象类型

November  23 , 2019
October  28 , 2020
  • 单片机
  • 数据类型
  • 数据结构
  • JavaScript

在JavaScript里目前提案的有6种基本类型和1种对象类型

这两种数据类型的区别为 :

基本类型的值是无法改变的

对象类型的值(存储地址)是可以改变的

基本类型例子1：

let a = 10;
let b = a;

console.log(typeof (a) + // number
    " " +
    typeof (b)); + // number


console.log(a); // 10
console.log(b); // 10

a = 18;

console.log(a); // 18
console.log(b); // 10

b = 33;

console.log(a); // 18
console.log(b); // 33

对象类型例子 ：

let c = {
    name: 'JavaScript',
    type: 'dynamic'
};
let d = c;

console.log(typeof (c) +
    " " +
    typeof (d)); // object object 

console.log(c.name + " " + c.type); // JavaScript dynamic
console.log(c.name + " " + d.type); // JavaScript dynamic

d.name = 'Python';

console.log(c.name + " " + c.type); // Python dynamic
console.log(d.name + " " + d.type); // Python dynamic

c = [12, 34, 56];

console.log(c); // 12,34,56
console.log(d.name + " " + d.type); // Python dynamic

从上边的基本类型例子和对象类型例子的对比可以看出原始类型的值没有改变，而对象类型的值会改变，原始类型存储的是值，对象类型存储的是地址

文字解析

let a = 10;

声明 a 变量，并赋值 10
值 10 是 number 类型也属于原始类型，所以在stack栈内存中储存一个栈地址名称为 a 值为 10

let b = a;

声明 b 变量，并赋值了 a 变量
注意! 因为 a 变量的值是原始类型，所以这里的 = 在遇到原始类型时仿有拷贝的作用，所以 b 拷贝了 a 的值，即b = 10 ，最后在stack栈内存中新储存了一个栈地址名称为 b 值为 10

a = 18;

a 变量赋值 18
因为 a 变量在这里是重新赋值，所以在stack栈内存中在又新储存一个栈地址名称为 a 值为 18
注意! 之前栈内存中名称为 a 的栈地址会被清除，而它储存的值 10 还会存在，它只是保留了旧的栈空间数据但它的栈地址名称返回为默认值了

b = 33;

与 a = 18 描述一样

let c = {
    name: 'JavaScript',
    type: 'dynamic'
};

声明 c 变量，并赋值了两个键值对的对象类型
c 变量储存的是对象类型，所以在heap堆内存空间中申请了一个堆地址，这个堆地址放入了 c 变量对象类型的内容，而在stack栈内存空间中也储存了一个名称为 c 地址为刚刚在heap堆内存中申请的堆地址，可以用指针来比喻stack栈内存中的 c 指向了这个堆地址
注意! 对象类型在stack栈中储存的是地址，而不是值，原始类型储存的才是值

let d = c;

声明 d 变量，并赋值了 c 变量
继续在stack栈内存中储存名称为 d 的栈名称，因为赋值了 c 变量对象类型，所以 d 储存的是heap堆地址，而储存的这个堆地址正是 c 变量储存的堆地址，可以理解成 c 和 d 一起绑定(指向)了这个堆地址

d.name = 'Python';

既 c变量 和 d变量绑定了同一个堆地址，这个堆地址储存了name: 'JavaScript', type: 'dynamic'
d变量 d.name = 'Python'，这里在属性上做了更改，把原来name: 'JavaScript'，更改成了Python>
从而这个对象在堆内存也跟随改变。
